Zekr FAQ
Frequently Asked Questions of the Zekr project.

[General]

What is Zekr?
Zekr is an open platform for working on the Holly Quran. For more details see a list of features for Zekr 1.0 plan here. You can also find answers to some of your questions about the Holly Quran here.
What does Zekr mean?
Zekr is a name for the Holly Quran (in Quran terms). Linguistically, it means "note", "reminder" or "message". More clearly, Quran is a reminder for the human morality.
Where is the homepage?
The main homepage for zekr is http://siahe.com/zekr, which is updated with the latest builds of Zekr. There is also a secondary home page on the leading hosting service for open source projects: sourceforge.net. The URL is http://zekr.sourceforge.net. It may not update as fast as siahe.com. Conventionally package naming is of the form net.sf.zekr.*.

Why are you working on Zekr?
There is not a really nice and standard platform for Quran related topics. There are some software which are fully commercial, others are free-commercial but no source code available. I couldn't find such standard and up to date platform. If you know one, tell us.

[Developer]

What is Zekr written in?
Zekr is written in Java (JDK 1.4+ compliant). It is developed in Eclipse platform (3.0 Mx to 3.11).
Which GUI library is used?
Zekr is fully developed on SWT. Since it is not included in the standard Sun's JDK, you need to have swt-win32-xxxx.dll for Win32 machines or libswt-gtk-xxxx.so/libswt-pi-gtk-xxxx. for your Linux/GTK platforms. There is also a platform specific jar file on the project's lib directory named swt-xxx.jar, where xxx can be win32 or gtk!
Where is the source code?
Latest Zekr source code may always be available on SourceForge.net CVS. Use this URL for web based CVS tree and this one for checkout instructions.
Why Java
Simply because I know Java more than C++!
In adition it is cross platform, easy and open. I used SWT because it is a nice looking native widget toolkit and has a number of advantages over Swing (More fast, cool, and easy to code).
Do you Linux?
Yes! Zekr can also be run on Linux machines (at least I tested it on OSS 10). You should first download and install a Mozilla GTK2 (1.4+) browser from mozilla.org. You can download Mozilla 1.7.3 GTK2 from Mozilla.org.
Please remember where the Mozilla in installed (e.g. /ust/local/mozilla), and check if there is a components folder inside the mozilla home directory. You should set an environment variable MOZILLA_FIVE_HOME to the mozilla home directory. If there is no components folder inside mozilla home, check [mozilla home]/lib, and if there was such a folder, set the environment variable to this location.
On my Open Source SuSE 10.0, mozilla is installed on /opt/mozilla, but the main folder is /opt/mozilla/lib where components folder can be found.

After all, you can run [zekr]/run.sh to launch the Zekr platform.
Other platforms
By now, there is no other supported platform except Win32 and Linux GTK.

[Install]

I downloaded Zekr. Now, what should I do?
Please note that there are two releases for each version of Zekr for any platforms. One is installer based version. If you have downloaded this file, you can easily follow the installation steps and having JRE 1.4.1+ installed on you computer, you can then run Zekr. There is also another version available as a zipped archive which is proper for IT professionals!
I've seen an XP style screenshot of Zekr, but my copy is not in XP theme. What should I do?
Note that installer will try to put manifest files at proper locations from 0.1.0 release and there is no need to manually do that. If it still seems like windows classic theme try the following:

It is mainly about the GUI toolkit used for Zekr (SWT). Zekr supports XP theme, but you should do something manually. Get ready? OK, let's go:
Make sure where the javaw.exe in your path environment variable is. Some possibilities are:
  • [Windows]/system32/javaw.exe
  • [JAVA_HOME]/bin/javaw.exe
  • [Program Files]/java/j2re1.4.x/bin/javaw.exe
You should copy the file javaw.exe.manifest (or java.exe.manifest - located at [zekr]/res - if you want to start the application with java.exe) near the java.exe and javaw.exe. That's all.
Which languages does Zekr support?
Zekr has Language Engine for supporting multiple languages. There are corresponding XML files for each language pack (located at [zekr]/res/lang). You can write your own language pack (e.g. [zekr]/res/lang/esperanto.xml) for Zekr, and add a new element to [zekr]/res/config/zekr-config.xml, indicating the new language pack. For example if you want to write a new language pack for Esperanto language, make esperanto.xml near other XML language packs, and change zekr-config.xml this way:
Before:
	<language default="fa" current="">
		<pack file="farsi.xml" id="fa_IR" name="فارسی" />
		<pack file="english.xml" id="en_US" name="English" />
	</language>
After:
	<language default="fa" current="sp">
		<pack file="farsi.xml" id="fa_IR" name="فارسی" />
		<pack file="english.xml" id="en_US" name="English" />
		<pack file="esperanto.xml" id="sp_XX" name="Esperanto" />
	</language>
Please note that when current is set to "" so, Zekr tries to find the language pack equal to user's system locale (user.language JVM property). If it fails, it will use the default language pack instead.
For now there are 6 language packs available (alphabetically): Arabic, English, Farsi (AKA Persian), French, German, and Indonesian.
